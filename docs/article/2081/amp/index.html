暗影精灵3 移动硬盘安装 Linux系统

最近在学Linux，之前虚拟机用着很不爽，正好笔记本换了个SSD，于是把换下的128G硬盘买了个硬盘盒搞了个移动硬盘，双硬盘+双系统 岂不是美滋滋。

一. 安装前资源下载

WIN10安装很简单，网上随便一搜结果多的是，这里就不叙述了。接下来说一下在安装Linux系统前我们需要做的一些准备工作（我选择的Linux是Ubuntu的）：

  1. 准备一个大于等于4G的硬盘；
  2. 下载Ubuntu镜像文件，我是去官网下载的：https://www.ubuntu.com/download/desktop；
  3. 下载UltraISO软碟通，官网：https://cn.ultraiso.net/xiazai.html；
  4. 下载DiskGenius，http://www.diskgenius.cn/download.php；

二. 安装前资源配置

  1. 制作U盘启动盘：

    （1）安装完成打开软碟通软件
    （2）【文件】-【打开】，打开下载好的Ubuntu镜像文件
    （3）选中用作启动盘的U盘
    （4）【启动】-【写入硬盘映像】（5）然后[写入]就OK了
    2. 接下来对移动硬盘进行处理：

    （1）打开DiskGenius。注：这软件在我电脑上经常卡顿，不知是我下载的版本有问题还是怎样，总之在处理过程中多多等待别心急并不会影响使用
    （2）因为之前这张盘我是用来装系统的，还没格式化，这里先将它格式化，如果之前有分区，需要把分区都删除才能格式化
    （3）看该硬盘是否为GUID格式，如果不是转一下
    （4）然后对硬盘进行分区，这里我是这样分的：/boot = 2G， /swap = 8G， / = 50G， /home = 剩余空间，可以根据不同的需求来进行分区，当然最简单的还是分一个 / 和一个swap。具体的操作就是点【新建分区】然后在出来的对话框中修改大小以及格式，格式的话除了swap选择 [Linux swap…] ，其他的都设置成 [EXT4] 即可。GPT不用区分主分区之类的，所以那一项不用管。
    因为我的硬盘全都分完了，所以接下来的图去官网随便找张做示范了。
    3. 分区完成，准备工作就已经全部OK了，接下来开始安装。

三. 安装Linux系统

由于昨天安装系统时忘了拍照，且安装并不是很麻烦，这里根据记忆进行叙述：

  1. 将U盘与移动硬盘都连接到电脑上，开机并按 [F9]，选择从U 盘进行启动；
  2. 然后选择第二项 [ Install Ubuntu ] ;
  3. 开始是一些常规设置，如语言、完全安装还是精简安装之类的。要注意到了安装类型那一步，选择最下面的 【其他选项】；
  4. 因为做准备工作时已经将移动硬盘进行了分区了，接下来就是找到那几个分区，进行挂载。记得没错的话是每点到一个分区下方会有个【更改】选项，只需要在弹出的对话框中将【用途】那一栏选一下就OK了，4个分区都要配置一下（这里是因为我的分区方案是分为4个）；
  5. 可以看到下方还有个引导设备安装位置的选项，这里我选择的是移动硬盘（关于这个选择有点小问题，后面会提到）；
  6. 到了这里可以说系统就等于装好了，点击安装，接下来还有些常规设置，需要注意的可能也就是别设置了密码自己回头忘掉。

四. 你以为这就结束了吗？hhhhhhhhh

系统安装后所遇问题汇总：

  1. 当你安装完成，输好密码，然后画面卡住不动了，该怎么办？

    这里我遇到了这种情况，因为暗影精灵3是双显卡，自带的N卡跟Ubuntu自带的显卡驱动存在一些冲突，所以每每输入密码准备登录时都会卡住。
    解决办法是在开机时选择recovery模式进入Ubuntu，然后参照这个帖子进行设置。https://blog.csdn.net/muyuxiaoxiang/article/details/80502703
    这里我在进入recovery模式有时会出现些莫名其妙的现象，然后通过 [Ctrl] +[ C ]就可以解决了。

  2. 第一个问题解决后终于可以一睹Ubuntu的庐山真面目，可是把玩了一会当你想上网时发现连不上无线，打开Wifi会提示“找不到适配器”，好不容易装个系统还不让上网，我当时立马去网上找解决办法，最后还是解决了。

    好像是因为暗影精灵3 的网卡的问题，在Linux下会跟系统某些底层组件起冲突。解决办法很简单，进入BIOS把【安全启动模式】关了，就OK了。

  3. 因为硬盘盒昨晚才到家，折腾完就12点了，想着应该没问题了，收拾收拾就躺了。但是今天起来开机屏幕是这样的我关了机重开就又好了。研究了半天，发现是因为引导是装在了移动硬盘里，我的电脑USB充电功能关了，所以第一次开机时移动硬盘没连上，于是引导出现了问题。随即我又试了下把移动硬盘拔了开机，果真也是这种情况。今后总不可能移动硬盘随时都跟电脑在一块吧，所以这个问题一定要解决掉。
    最后通过浏览了很多帖子以及大量的试验，终于找到了解决之道。在不影响两个系统使用且不需要继续折腾的情况下，不用去做多余的工作就好了。而当移动硬盘不在的情况下，开机出现上图的现象，只需要在 Grub > 的后面键入 exit，就可以正常启动Win系统了。
