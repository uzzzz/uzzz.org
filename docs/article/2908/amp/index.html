移动硬盘安装ubuntu全程纪实（折磨了我整整两天）完美实现

fandyvon
10 years ago

ubuntu-9.04-desktop-i386.iso：这多说了，这个必须有
grub4dos-0.4.4.zip：只用到压缩包里的两个文件grldr和menu.lst
宽带：很重要，要是不能上网更新，用CD版装完的Ubuntu对新手来说，用起来太困难，带宽越高越好，我用的是局域网共享方式上网(2M)，所 以后面关于网络设置的部分和我方式不同的，请参考其它贴子。

开始安装

1.移动硬盘准备两个分区，一个是安装ubuntu系统的，还有一个是交换分区，交换分区不要太大，一般设置为内存的两倍即可。

2.ubuntu-9.04-desktop-i386.iso复制到C盘根目录 下。

3.用winrar打开ubuntu-9.04-desktop-i386.iso 文件，然后将里面的casper文件夹解压到C盘根目录下。

4.打开C盘下的casper文件夹，将initrd.gz和vmlinuz文件复制到C盘根目录下。

5.用winrar打开grub4dos-0.4.4.zip文件，将grldr和menu.lst文件解压到C盘根目录下。

6.编辑menu.lst，最好使用EditPlus这相的纯文本编辑工具打开，如果没有，用写字板也行，不要用记事本或word；
将下面内容添加到menu.lst的最后

title install Ubuntu 9.04
root (hd0,0)
kernel /vmlinuz boot=casper iso-scan/filename=/ubuntu-9.04-desktop-i386.iso
initrd /initrd.gz

7.去除C盘下boot.ini的只读，系统，隐藏属性，方法如下
开始->运行->输入：attrib -r -h -s c:/boot.ini

8.打开C盘根目录下的boot.ini文件
修改timeout=5
在最后添加一行 c:/grldr=”install Ubuntu 9.04”

好，windows下的操作完成。

9.重启电脑，在启动菜单里选install Ubuntu 9.04然后在出现的 菜单里再选install Ubuntu 9.04。

10.进入图形界面双击install。

11.语言选择中文(简体)，然后Forward

12.时间默认Shanghai，直接Forward

13.键盘布局默认USA，直接Forward

14.然后出现一堆英文，大概意思是问你是不是使用现有的硬盘分区结构来安装，选NO

15.选择手动指定分区(高级)，然后Forward

16.建立交换分区和主分区
1.指定主分区，格式为ext3
2.指定交换分区

4.然后Forward

17.输入你的身份信息，然后Forward

18.迁移文档和设置，什么也不选，直接Forward

19.这里选择高级选项，里面指定启动项的位置，选择到你的移动硬盘安装目录，如果不选将 导致XP系统的MBR改写，即只有插入移动硬盘才能启动XP系统。

20.

会出现一个关于/isodevice的弹出框，处理方法
Ctrl+Alt+F2，输入sudo umount -l /isodevice回车，然后Ctrl+Alt+F7，点继续
这时会回到分区那步，一路下一步，就可以安装了。

等，然后重启吧，不知道是不是所有人都一样，我在黑白界面上要再按一次回车，才会生启

再开机就是漂亮的Ubuntu登录界面了，可惜好多E文

汉化

1.上网：我用的是局域网，修改IP就可以上网，方法是
1.system->preferences->Network Connections
2.选择Auto etho->Edit
3.选择IPv4 Settings
4.Method修改成Manual
5.Add->输入IP地址(Address),子网掩码(Metmask),网关(Gateway),DNS,然后Apply

2.下载安装中文包及中文支持软件
1.system->Administration->Language Support
2.会弹出一个对话框，然后Update
3.又一个对话框，点install

等，然后重启，你就可以体验ubuntu带给你的新鲜和快感了。

第一次安装在第19步时未选择而直接覆盖了电脑上的硬盘，导致拔掉硬盘后无法启动windows（grub命令可以进去windows：用set root=(hd0,0)等参考grub2教程）。拔掉移动硬盘后出现：grub rescure，但此时grub大部分命令也都不可用，因为其grub核心也在移动硬盘里。

解决办法：

将grub重新装载到移动硬盘的mbr里，然后恢复windows的mbr

开始时候用mbrfix工具，和fdisk /mbr都不见效果，修复几次后情况越来越糟糕，开始是 grub can’t find disk;最后发展为找不到硬盘，出现disk error。

1.最后用dos工具箱里华顿医生，将mbr修复

2.修复后发现除了c盘以外的分区丢失，之后用分区表修复工具修复

如嫌将grub重新装载到移动硬盘的mbr里比较麻烦（需要在linux下用grub shell命令实现），可以按上述20步重新安装，19步特别注意装载到你的移动硬盘里。

之后在启动的时候在bios里进行设置下，选择哪个盘进行启动，你主机的硬盘还是你的移动硬盘为第一启动盘。（本启动方法对将grub写入mrb里的起作用，未写入的可以用grub命令加载ubuntu核心进行启动）
